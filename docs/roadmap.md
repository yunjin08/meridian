## UI / UX gaps (Phase 1 incomplete work)

### 4. Bollinger Bands overlay on chart

The BB values are calculated and shown in `IndicatorPanel` as numbers, but they are not drawn as lines on the candlestick chart itself. A proper BB overlay requires three `LineSeries` instances (upper, middle, lower) added to the same chart instance in `ChartContainer`.

**How:** In `ChartContainer.tsx`, add three `LineSeries` refs after the candlestick series. In the candle update effect, call `lineSeries.setData()` with the BB arrays mapped from indicator data.

### 5. RSI and MACD subcharts

Currently RSI and MACD values are displayed as text only. Most trading dashboards render them as subcharts below the main candlestick chart.

**How:** TradingView Lightweight Charts supports multiple panes. Create a second chart instance in its own `div` below `ChartContainer`, or use a single chart with `PriceScaleId` to separate panes.

### 6. Volume bars

Candle volume is fetched but never rendered. Add a `HistogramSeries` to `ChartContainer` for volume bars at the bottom of the chart.

### 7. Reconnect button

When `connectionStatus === 'failed'` (after 5 WS reconnect attempts), the UI shows the OFFLINE badge but there is no manual "Reconnect" button. Add one to `ConnectionStatus.tsx` that calls a function exposed from `useBinanceWebSocket`.

**How:** Expose a `reconnect()` callback from the hook (via a ref or a store action) and wire it to a button in `ConnectionStatus`.

### 8. Balance refresh button

`BalanceCard` shows the last-updated timestamp but has no manual refresh button. Add one that calls the `load()` function exposed from `useBalance`.

## Feature additions (Phase 2)

### 9. Background alert notifications via Telegram or Pushover

Currently alerts only fire while the tab is open. The fix: when an alert triggers client-side, call a new Netlify Function (`/api/notify`) that posts to a Telegram bot or Pushover API. This delivers the notification even if the tab is later closed.

**How:**
1. Create `netlify/functions/notify.ts` â€” accepts `{ title, body }`, calls Telegram `sendMessage` or Pushover API.
2. Add `TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID` (or `PUSHOVER_USER_KEY` + `PUSHOVER_APP_TOKEN`) to env.
3. In `useAlertEvaluator.ts`, after `sendNotification()`, also `fetch('/api/notify', { method: 'POST', body: ... })`.

### 10. Multi-symbol support

Symbol is hardcoded to `BTCUSDT` in `constants.ts`. Add a symbol selector to switch between ETH/BTC/SOL etc.

**How:**
- Add `activeSymbol` to `chartStore`.
- Pass it to `useCandles`, `useBalance` (filter different assets), and `useBinanceWebSocket` (subscribe to different streams).
- Add a symbol selector component to the header.

### 11. Order book / depth panel

Add a real-time order book panel using the `btcusdt@depth` WebSocket stream. Display top 10 bids and asks.

**How:** Subscribe to the depth stream in `useBinanceWebSocket` and add a new `orderbookStore` + `OrderBookPanel` component.

### 12. Trade history

Show recent personal trades from Binance via `GET /api/v3/myTrades`. Add a new Netlify Function `trades.ts` and a `TradeHistory` component.

### 13. Price alerts via threshold drawing on chart

Let the user click on the chart to set a horizontal price line, which automatically creates a `price_crosses` alert at that level. Requires `chart.subscribeClick()` from the TradingView API.

## Technical debt / code quality

### 14. Remove unused files from Vite template

These files were generated by the Vite template and are no longer used:
- `src/App.css` â€” styles removed from `App.tsx`, file is still present
- `src/assets/react.svg`, `src/assets/vite.svg`, `src/assets/hero.png`

### 15. `_indicators.ts` has dead code

The `highs` and `lows` variables are computed but then suppressed with `void highs; void lows`. They were intended for future ATR or Stochastic calculations. Either use them or remove them.

### 16. ESLint is installed but not configured for this project

`eslint.config.js` was generated by Vite. It should be updated to include:
- `@typescript-eslint/strict` rule set
- `eslint-plugin-react-hooks` rules (enforces correct `useEffect` dependency arrays)
- Remove the `// eslint-disable-line react-hooks/exhaustive-deps` suppressions in hooks once deps are properly specified.

### 17. Error boundaries

No React Error Boundary is in place. If `ChartContainer` throws (e.g. TradingView API issue), the whole app crashes silently. Add an `ErrorBoundary` wrapper around `ChartContainer` and the dashboard.

### 18. Loading state race condition in `useCandles`

If the user switches timeframes rapidly, the debounce prevents excess fetches, but a slow in-flight request could resolve after a newer one and overwrite fresher data. Fix: use an `AbortController` tied to the effect cleanup to cancel in-flight fetches on timeframe change.

### 19. `useBinanceWebSocket` subscribe/unsubscribe on timeframe change

The kline stream subscribe/unsubscribe logic runs in a `useEffect` that watches `activeInterval`. If the WS is reconnecting when the timeframe changes, the new stream subscription is never sent. Fix: on reconnect success, re-subscribe to the current interval's kline stream.
